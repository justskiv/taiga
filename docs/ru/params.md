# Параметры конфигурации

Всё, что читает тема, лежит под `[params]` в `hugo.toml` сайта. Задавай только
то, что нужно, — у каждого ключа есть разумный дефолт (указан ниже), а
остальное тема достроит сама. Палитры здесь **не** настраиваются — это файлы
данных (см. [customizing.md](customizing.md#palettes-are-data)).

Рабочий минимальный конфиг лежит в
[`exampleSite/hugo.toml`](../../exampleSite/hugo.toml), полностью
закомментирован — скопируй и правь под себя.

## Параметры сайта

| Ключ | Тип | Дефолт | Что делает |
|---|---|---|---|
| `brand.name` | строка | `"taiga"` | Вордмарк в шапке и подвале. Дефолт задан в собственном `hugo.toml` темы, а не фолбэком в шаблоне — переопредели его в конфиге своего сайта. |
| `brand.tld` | строка | `""` | Необязательный суффикс вордмарка, например `".dev"`. Пусто = только имя. |
| `heroLine` | строка | — | Строка героя на главной (смысловая половина). Общий для сайта параметр, если язык его не переопределяет — см. примечание 2. |
| `heroSat` | строка | — | Хвостовое « · …»-уточнение строки героя, оформленное отдельно. Общий для сайта, если язык его не переопределяет — см. примечание 2. |
| `featured` | строка (путь к странице) | — | Карточка «гайд недели» на главной, например `"howto/get-started"`. Не задан — карточки нет. Общий для языков — путь резолвится в перевод страницы на каждом языке. |
| `feedInitial` | int | `8` | Сколько постов видно в ленте на главной до кнопки «ещё N гайдов». |
| `defaultTheme` | строка (id палитры) | `"taiga"` | Палитра, применяемая до того, как читатель выберет свою, — id файла `data/themes/<id>.toml`. |
| `accent` | строка (`#rrggbb`) | — | Переопределяет цвет акцента сразу во **всех** палитрах (встроенных и сайта) — ось бренда, ортогональная палитрам. Не задан ⇒ каждая палитра остаётся при своём акценте. Обязателен 6-значный hex, например `"#7aa2f7"`; любое другое значение валит сборку. См. [customizing.md](customizing.md#recolour). |
| `accentDim` | CSS-цвет | вычисляется | Переопределение приглушённого акцента (кольца фокуса, подсветка заливок). Дефолт при заданном `accent`: `rgba(r,g,b,.18)`, выведенный из него. Игнорируется, если `accent` не задан. |
| `accentGlow` | CSS-цвет | вычисляется | Переопределение свечения акцента (тени). Дефолт при заданном `accent`: `rgba(r,g,b,.28)`, выведенный из него. Игнорируется, если `accent` не задан. |
| `versionDefault` | строка | — | Фолбэк для чипа версии гайда, если front matter не задаёт `version`. Произвольная строка («go1.26», «PostgreSQL 17») — см. примечание 3. Если не задано ни то, ни другое, чип не выводится вовсе. |
| `linkcheck` | `"error"` \| `"warn"` | `"error"` | Битая внутренняя ссылка валит сборку (`error`) или просто пишет предупреждение (`warn`). |
| `ogImages.enable` | bool | `true` | Генерировать обложки Open Graph во время сборки. |
| `ogImages.style` | строка (папка) | `"dots"` | Стиль обложки = имя папки под `assets/og/` (см. [customizing.md](customizing.md#og-cover-styles)). |
| `rss.fullContent` | bool | `false` | Класть в RSS-фид полный текст гайда, а не сокращение — см. примечание 1. |
| `footerTag` | строка | по умолчанию `heroLine` | Тэглайн в подвале. Общий для сайта, если язык его не переопределяет — см. примечание 2. |
| `social.twitter` | строка | `""` | `@handle` для `twitter:site`. Пусто ⇒ тег опущен. |
| `rubricSections` | список строк | — | Какие секции контента — **рубрики**: их листовые страницы получают полный layout гайда (два рельса, kicker, meta, TOC, мост между сериями). Все прочие страницы получают простую колонку. Также определяет карточки рубрик на главной и список рубрик на 404. |

**Примечания**

1. `exampleSite` задаёт `rss.fullContent = true`; без значения фид падает
   обратно на сокращения.
2. `heroLine`, `heroSat` и `footerTag` — параметры уровня сайта, но в
   многоязычном сайте они естественно читаются как текст конкретного
   языка — `exampleSite` задаёт все три внутри `[languages.en.params]` и
   `[languages.ru.params]`, а не на верхнем уровне. Всё остальное в этой
   таблице (`defaultTheme`, `featured`, `rubricSections`, …) общее для языков,
   пока языковой блок его не переопределит.
3. `exampleSite` задаёт `versionDefault = "v0.0.1"` — собственную версию
   темы; заодно это живой пример произвольной строки.

## Режим «скоро открытие»

Сайту, на котором ещё нечего публиковать, всё равно нужен живой домен, TLS и
работающий деплой — чтобы день запуска был переключением флага, а не
инфраструктурным проектом. `comingSoon.enable` заставляет тему отвечать **на
любой маршрут** одной самостоятельной карточкой: логотип и короткое «скоро» —
и больше ничего.

| Ключ | Тип | По умолчанию | Что делает |
|---|---|---|---|
| `comingSoon.enable` | bool | `false` | Включает режим. |
| `comingSoon.title` | строка | `site.Title` | `<title>` и `og:title`. |
| `comingSoon.description` | строка | `heroLine` | Meta и `og:description`. |
| `comingSoon.words` | список строк | `["Soon", "Скоро", "近日公開"]` | Строка «скоро», по слову на язык. Третье рендерится CJK-стеком шрифтов. |
| `comingSoon.foot` | строка | — | Приглушённая строка под карточкой. Опустите, чтобы скрыть. |

Ветвление стоит в `baseof.html`, **выше** `head.html`, — поэтому в этом режиме
CSS-бандл, модалка поиска, меню и OG-граф не *собираются* вовсе, а не
вырезаются постфактум. В опубликованном исходнике лежит карточка и ни намёка на
сайт, который пишется за ней. Карточка переиспользует `logo.html`,
`head/fonts.html` и `hooks/head-extra.html`, так что у логотипа, шрифтов и
фавикона не появляется второго источника правды.

Сочетайте с `disableKinds`, чтобы Hugo даже не генерировал остальные страницы:
тогда гайд, случайно потерявший `draft: true`, всё равно не доедет до публичной
сборки. Держите оба ключа в оверлей-конфиге, который обычные команды не грузят,
— и `hugo server -D` продолжит показывать черновики, пока вы пишете:

```toml
# hugo.stub.toml — подключается только так: hugo --config hugo.toml,hugo.stub.toml
disableKinds = ["page", "section", "taxonomy", "term", "rss", "sitemap"]
[outputs]
  home = ["html"]
[params]
  [params.comingSoon]
    enable = true
```

`robots.txt` сам переключается на `Disallow: /` — краулить нечего, а пустая
страница, проиндексированная сегодня, завтра станет плохим первым впечатлением.

## Это не параметры

- **Переключатель языка** (`EN · RU` в шапке) не принимает параметр. Он
  появляется сам, когда у сайта больше одного языка, и не рендерит ничего на
  одноязычном сайте — настраивать тут нечего.
- **Палитры** — файлы данных, по одной на палитру (`data/themes/<id>.toml`),
  а не параметры. Добавляй, переопределяй или отключай их файлом — см.
  [customizing.md](customizing.md#palettes-are-data). `defaultTheme` выше
  только называет, какая палитра грузится первой.
- **Данные роадмапа** — тоже не параметр. Это `data/roadmap.toml` (один
  язык) или папка языковых файлов `data/roadmap/<lang>.toml`, которые читают
  шорткод `roadmap` и полоса «в работе» на главной.
- **Меню** — это стандартные Hugo `[[menus.main]]` / `[[menus.footer]]`.
  Пункт меню рисует маленький значок `<sup>`, если у него задан
  `params.badge` (например, `"β"`) — обычная функция навигации, в демо
  использована для раздела «Песочница».
- **Поиск** (модалка ⌘K + Pagefind) всегда включён; флага отключения нет.
  Он тихо деградирует в подсказку «индекс ещё не собран», если индекса
  Pagefind нет (обычный `hugo server` без второго шага сборки).
