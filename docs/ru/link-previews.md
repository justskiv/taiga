# Превью ссылок

На точном курсоре ссылка в тексте по наведению растит плавающую **карточку-превью**
— физика терм-карточки, один поповер на страницу. Пять жанров, одна семья:

- **Внутренние** ссылки (на гайд или серию) открывают окно в страницу: шапка
  статьи в миниатюре, зачин плюс целые H2-разделы до бюджета слов, список «дальше
  в статье» и футер «Читать статью». Ссылка с `#якорем` сужает карточку до одного
  раздела; ссылка на лендинг серии показывает оглавление.
- **Telegram**-ссылки на пост из *твоего* канала открывают карточку, собранную из
  t.me на билде — аватар, имя, число подписчиков, текст поста (с фото, цитатами
  реплаев, реакциями и кастомными эмодзи), просмотры и дату.
- **YouTube**-ссылки открывают видео-карточку — превью-кадр (ссылка на ролик),
  длительность, канал, число просмотров.
- **Wikipedia**-ссылки (`*.wikipedia.org/wiki/*`) открывают карточку-сводку —
  serif-значок W, заголовок статьи, короткое описание и лид-extract.
- **Блог Go**-ссылки (`go.dev/blog/<slug>`, легаси `blog.golang.org/<slug>`
  нормализуется к нему) открывают карточку с курсивным значком *Go*, заголовком
  поста, байлайном (дата локализуется) и зачином.

Каждая внешняя ссылка получает значок-классификатор: ↗ куда-то в веб,
✈ твой Telegram, ▶ твой YouTube, W Wikipedia, *Go* блог Go. Внутренние ссылки —
без значка.
В хроме компонентов (шапка fold, подпись callout, CTA-карточка) значок гасится,
чтобы не дублировать собственную иконку компонента — карточка по ховеру там
работает, глушится только сам значок.

Фича **выключена** на тач-устройствах (тап обязан вести по ссылке) и уважает
`prefers-reduced-motion` (только fade, без шиммера).

## Физика ховера

Ритм общий с терм-карточками: **250 мс** намерения до открытия (`OPEN_DELAY`
в `term.js` использует то же значение — случайный проезд курсора ничего не
открывает), **120 мс** грейса на закрытие с перепроверкой `:hover`, и
«тёплое» окно ~450 мс после закрытия, в котором следующая карточка
открывается почти сразу. Вокруг поповера — невидимый **ореол 12 px**: он
мостит зазор до ссылки-якоря и прощает промахи курсора у краёв (терм-карточки
остаются на временно́м грейсе — их `overflow:hidden` клипает любой ореол).
Карточки следуют за скроллом и ресайзом, открываются вверх, когда снизу нет
места, и закрываются по Escape.

## Как включить

```toml
[params.linkPreviews]
  enable = true          # по умолчанию выключено
  budgetWords = 1400     # сколько статьи показывать в превью, целыми H2

[params.telegram]
  channels = ["your_handle"]   # ТВОИ каналы — их посты t.me получают карточку
```

Внутренние превью отдаются как **вторичный output-формат** — голый фрагмент по
адресу `/<slug>/index.preview.html`. Тема объявляет формат `preview` и включает
его для `page` и `section`. **Если на сайте задан свой `[outputs]`**, Hugo
перекрывает этот ключ карты поэлементно (не дополняет), поэтому `preview` нужно
повторить там:

```toml
[outputs]
  page    = ["html", "preview"]
  section = ["html", "preview"]   # для карточек-оглавлений серий
  # home / taxonomy / term превью не нужен
```

Пост t.me из канала, которого **нет** в `telegram.channels`, получает только
значок ✈, без карточки. Любая ссылка `youtube.com/watch` или `youtu.be` получает
карточку.

## Что тянется на билде

Карточки Telegram и YouTube собираются через `resources.GetRemote` во время
сборки — из браузера в t.me/YouTube **не** ходит ничего (у t.me и так закрыт
CORS). Билд:

- парсит `t.me/<ch>` (имя, подписчики, описание, аватар) и
  `t.me/<ch>/<id>?embed=1` (текст, фото, просмотры, дата, реплай, реакции,
  кастомные эмодзи), скачивая всё медиа — CDN-URL телеги живут с истекающими
  токенами;
- читает инлайн-`videoDetails` со страницы YouTube (без API-ключа) и качает
  превью-кадр (`maxresdefault`, фолбэк `mqdefault`);
- берёт официальный REST-JSON сводки Wikipedia
  (`<lang>.wikipedia.org/api/rest_v1/page/summary/<title>?redirect=true`) — тот
  же эндпоинт, что у собственных превью Википедии, поэтому никаких регексов.
  **Wikimedia требует описательный `User-Agent`**, иначе отвечает `403`; тема
  шлёт его, собирая из `baseURL`. Кириллические тайтлы percent-энкодятся. Текст
  под **CC BY-SA** — атрибуцию несёт сама карточка, ссылаясь на статью. Страница
  типа `disambiguation` или без extract карточки не даёт (ссылка остаётся обычной
  внешней со значком W), а длинное автогенерённое `description` (частое на
  ru.wikipedia) отбрасывается, чтобы не дублировать extract.
- парсит страницу поста **блога Go**: заголовок (первый `<h1>` без класса),
  байлайн `<p class="author">` (авторы + дата, дата локализуется под язык
  страницы) и зачин из контейнера `<div class='markdown'>` — верхнеуровневые
  блоки до первого `<h2>`, а если пост начинается сразу с `<h2>` — блоки после
  него. Нет `<h1>` (или фетч упал) → карточки нет, ссылка остаётся внешней со
  значком *Go*.

Запросы кэшируются через `partialCached` по URL — каждый пост/видео тянется
**один раз** за билд, а Hugo кэширует ответы на диск между сборками.

### Деградации (билд никогда не падает)

Каждый запрос обёрнут так, что сетевая ошибка или недоступный ресурс приводят к
деградации, а не к падению сборки — **офлайн-билд всё равно проходит**:

- пост t.me, который не спарсился (удалён, приватный или создан после последнего
  билда) → **карточка канала** (аватар, имя, подписчики, описание);
- канал, который не удалось получить → минимальная карточка, всё ещё называющая
  канал;
- страница YouTube без `videoDetails` → **без карточки**, ссылка остаётся обычной
  внешней;
- кастомный эмодзи, чей арт не скачался → показывается его юникод-глиф.

Раз карточки печёт билд, счётчики поста освежаются каждым деплоем, а единственная
дыра — пост, созданный после последней сборки — закрыта карточкой канала.

## Что показывает превью статьи

Фрагмент превью несёт *всю* статью, нарезанную на разделы. JS показывает зачин и
целые H2-разделы, пока суммарный текст не достигнет `budgetWords` (минимум один
раздел), затем перечисляет оставшиеся H2 под «дальше в статье». Интерактив
нейтрализуется на билде, чтобы превью осталось тихим окном для чтения:

- фигура `{{</* widget */>}}` → карточка-плейсхолдер (⚙ + подпись);
- codapi-сниппет → блок кода `<pre>` остаётся, за ним тихий бейдж «запускается в
  статье»;
- любые `<script>` / `<style>` / `<iframe>` во фрагменте → вырезаются.

Callout'ы, fold'ы, таблицы и картинки остаются как есть.

## Область

Превью получают только ссылки в тексте статьи; оглавление, сноски, мост серии и
рельсы исключены. Ссылка на текущую страницу не превьюится.
