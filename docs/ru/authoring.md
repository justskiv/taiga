# Как писать гайд

Гайд — это **папка-бандл**: каталог с `index.md` и, если в тексте есть живые
иллюстрации, соседней папкой `widgets/`. Эта страница — контракт на то, как его
написать: front matter, шорткоды, код, ссылки, диаграммы, виджеты, обложки,
переводы и голос.

## Начни с архетипа {#archetype}

```sh
hugo new content howto/my-guide --kind guides            # одиночный гайд
hugo new content howto/handbook/my-guide --kind guides   # часть серии «handbook»
```

Получишь каркас бандла — `index.md` с заполненными основными полями и заготовку
`widgets/w-example/widget.js`. **В пути нет `/index.md`**: передавай папку
бандла, иначе Hugo переключится в режим одиночного файла и молча не скопирует
соседние файлы архетипа. Папка без `widgets/` — первый признак этой ошибки.

Первый сегмент пути должен быть секцией из `params.rubricSections` — именно это
делает страницу гайдом (две панели, кикер, чипы меты, оглавление, мост серии),
а не обычной колонкой. Гайд, положенный на одну папку глубже, — **часть
серии**, см. [Серии](#series-and-weight).

## Front matter {#front-matter}

```yaml
title: "Значения и layout: как Go-значение лежит в памяти"
slug: memory-1-layout        # фиксирует URL (…/memory-1-layout/)
date: 2026-03-07             # лента, RSS, sitemap — в тексте статьи не появляется
description: "Одно предложение — для ленты, поиска и meta."
lead: "Лид-абзац (может быть длиннее description)."
weight: 1                    # порядок части внутри серии; у одиночного гайда — опустить
tags: [memory, layout, unsafe]
mins: 12                     # чип «~12 мин» — руками, а не .ReadingTime
version: "go1.26"            # свободный чип «проверено на»; без него — versionDefault
rail_title: "Layout"         # необязательно: 2–4 слова для левой панели и миникарты
related: []                  # для одиночных гайдов: 3–5 контент-путей для панели «ещё по теме»
```

| Поле | Обязательно | Что делает |
|---|---|---|
| `title` | да | Режется по первому `": "` на `<h1>` и `.sub`; переопределяется через `sub:`, а заголовок без `": "` остаётся в одну строку. |
| `slug` | да | Фиксирует красивый URL; после публикации не меняй — и держи **одинаковым во всех переводах** (см. [Перевод гайда](#translating-a-guide)). |
| `date` | да | Порядок ленты и RSS, печатается на карточке в ленте. **В тексте статьи не появляется никогда.** |
| `description` | да | Карточка ленты, результат поиска, `og:description`, `<description>` в RSS. |
| `lead` | желательно | Вводный абзац. Он же запасной вариант для `og:description`. |
| `weight` | для частей серии | Порядок части внутри папки серии. См. [Серии](#series-and-weight). |
| `tags` | желательно | Чипы, ведущие на страницу тегов, на якорь `#<tag>`. Держи **одинаковыми во всех переводах**. |
| `mins` | желательно | Чип времени чтения, сумма длительности на странице рубрики и шкала моста серии; без него — `.ReadingTime`. |
| `version` | опционально | Свободный чип «проверено на», печатается как есть (`go1.26`, `PostgreSQL 17`). Без него — `params.versionDefault`; нет ни того, ни другого ⇒ чипа нет. |
| `arch` | опционально | Дописывается к чипу версии через `·`, например `64-бит`. |
| `interactive` | опционально | Добавляет третий чип меты — перечень того, что на странице интерактивно. |
| `interactive_label` | опционально | Подпись этого чипа; по умолчанию — i18n-ключ `meta_interactive`. |
| `rail_title` | опционально | Короткое имя для левой панели, миникарты и моста серии. По умолчанию — заголовок до первого `:`. |
| `linkTitle` | опционально | Короткий заголовок для списка частей на странице рубрики. `<h1>` и `<title>` не трогает. |
| `related` | только одиночные | 3–5 **контент-путей** (`inside/anatomy/palette-is-a-file`, а не URL) для панели «ещё по теме». У гайда в серии игнорируется — там панель показывает части. |
| `placeholder` | опционально | См. [Заглушки](#placeholders). |
| `og_image`, `og_style` | опционально | См. [Обложки (OG-картинки)](#covers-og-images). |
| `sub` | опционально | Переопределяет автоматическое деление `title`. |
| `intro` | опционально | Markdown между чипами меты и оглавлением. |
| `foot` | опционально | Markdown в самом низу, под мостом серии. |
| `toc_labels` | опционально | Карта «id заголовка → короткая подпись», чтобы укоротить длинный `h2` в оглавлении: `toc_labels: {two-fields: "Два поля"}`. |
| `draft` | опционально | Штатный Hugo. Архетип ставит `draft: true`. |

`kicker:` и `headline:` читаются только на **не-гайдовых** страницах (о сайте,
роадмап, `/tags/`); у гайда кикер собирается из рубрики и серии, и задавать их
бессмысленно.

### Серии {#series-and-weight}

Серия — это **папка**, под-секция своей рубрики. Членство — то, где лежит
бандл, и ничего больше:

```
content/howto/
  handbook/                 ← серия
    _index.md               ← метаданные серии (ниже)
    get-started/index.md    ← часть, weight: 1
    first-guide/index.md    ← часть, weight: 2
  build-magic/index.md      ← одиночный гайд (прямой ребёнок рубрики)
  standalone/               ← необязательный контейнер одиночных гайдов (ниже)
    _index.md               ← params.standalone: true
    build-magic/index.md    ← одиночный гайд, идентичный верхнему
```

`weight` во front matter части — её порядок внутри серии. Кикер «часть N из M»,
левая панель и нижний мост (сегменты по `mins`, «осталось ~N мин») выводятся из
папки, на сборке. Добавить часть — положить бандл в папку серии; перенести гайд
в серию или из неё — это `mv`. Серия из одной части рендерится как одиночный
гайд: механика включается начиная с двух.

Каждая серия рендерит **лендинг** (`/<рубрика>/<серия>/`): заголовок, тэглайн,
описание, тело `_index.md` как эпиграф, список частей и кнопку «начать серию».
Страница рубрики показывает ту же серию блоком с якорем
(`/<рубрика>/#<серия>`); серия, анонсированная `_index.md` без частей,
появляется там тизером «готовится», а её лендинг рендерит анонс. Каркас
`_index.md` создаёт архетип:

```sh
hugo new content howto/handbook/_index.md --kind series
```

```yaml
title: "Память"             # имя серии: кикеры, блок на рубрике и лендинг
description: "Одна строка для блока серии на странице рубрики и лендинга."
weight: 10                  # порядок серий на странице рубрики
params:
  label: "память"           # короткое имя в кикерах (иначе — title в нижнем регистре)
  tagline: "3 части, от байта до GC"   # необязательная строка рядом с «серия ·»
```

### Одиночные гайды и контейнеры {#standalone}

Одиночный гайд — прямой ребёнок своей рубрики: без папки серии и без `weight`.
Когда одиночных становится много и папки серий тонут среди них, сложите их в
**контейнер** — под-секцию, чей `_index.md` состоит из одного маркера. Обе
формы равноправны, так что переезжать можно по одному `mv` за раз:

```yaml
title: "Standalone"         # нигде не рендерится
params:
  standalone: true          # «мои дети — одиночные гайды, я не серия»
build:
  render: never             # у контейнера нет собственной страницы
  list: local               # виден шаблонам рубрики, невидим глобально
```

Контейнер держит **только leaf-бандлы** — серию внутрь него класть нельзя. На
многоязычном сайте маркерный `_index.md` обязан существовать **в каждом языке**
(`_index.md` + `_index.ru.md`), иначе дерево другого языка сочтёт папку серией.

Гайдам **вне рубрик** заводится собственная секция верхнего уровня, указанная в
`params.extraGuideSections` (см. [params.md](params.md)): её страницы попадают
в ленту, поиск и RSS с полным статейным layout'ом, а кикер берёт `label` из
`_index.md` секции — но рубрикой она не считается (ни карточки на главной, ни
строки в 404).

### Заглушки {#placeholders}

Ненаписанный гайд можно выпустить заглушкой: полный front matter (лента живёт с
него), тело — одна строка, и `placeholder: true`. Такой гайд считается в ленте,
в облаке тегов и в структуре серии («часть N из M»), но **не попадает ни в RSS,
ни в поисковый индекс**. Дописал текст — снял флаг.

## Шорткоды {#shortcodes}

Их семь, и это весь набор.

### callout — `{{</* callout type="trap" */>}}` {#callout}

Пять типов: `key` (главная мысль), `trap` (ловушка), `note` (историческая
сноска), `internals` (под капотом), `warn` (внимание). Подпись берётся из i18n
(`callout_key`, `callout_trap`, `callout_note`, `callout_internals`,
`callout_warn`), переопределяется через `label=`. Тело — Markdown. По умолчанию
`type` — `note`.

```md
{{</* callout type="key" */>}}
Заголовок слайса — три слова: указатель, длина, ёмкость.
{{</* /callout */>}}
```

### fold — `{{</* fold icon="video" title="…" */>}}` {#fold}

Тихая сворачиваемая врезка. В свёрнутом виде — одна строка: значок, заголовок
и переключатель «Подробнее ⌄», без рамки и фона, так что читается как часть
текста; клик плавно раскрывает панель по высоте. Годится для отступления,
которое читатель может взять или пропустить: видео-версия, оговорка, полный
листинг. Тело — Markdown, а `title=` — инлайновый Markdown, поэтому ссылка в
заголовке работает и по клику не сворачивает панель.

```md
{{</* fold icon="video" title="Есть также [видео-версия](https://youtu.be/…)." */>}}
Сначала посмотрите, потом прочитайте для закрепления — что-то нагляднее на
экране, что-то лучше усваивается текстом.
{{</* /fold */>}}
```

| Параметр | Значение |
|---|---|
| `title=` | строка-заголовок, инлайновый Markdown (обязателен) |
| `icon=` | ведущий значок из набора ниже; по умолчанию `info`, `none` — убрать |
| `more=` | подпись в свёрнутом виде (по умолчанию `Подробнее`) |
| `less=` | подпись в раскрытом виде (по умолчанию `Свернуть`) |
| `open=` | `true` — отрендерить сразу раскрытым |

Значки: `video` · `info` · `tip` · `book` · `code` · `terminal` · `warning` ·
`star` · `link` · `note`.

Панель анимирует высоту — настоящий выезд, не фейд — через `interpolate-size`
на `:root` (задан в `00-tokens.css`); где браузер его не поддерживает, панель
раскрывается мгновенно. Подписи по умолчанию русские; переопределяются в
вызове.

### term — `{{</* term "mcache" */>}}` {#term}

Слово в тексте, за которым стоит полноценная карточка-определение: наведение
раскрывает её, клик закрепляет — можно спокойно читать и уйти по ссылке, Escape
закрывает. Тело — Markdown: жирный, инлайн-код, списки, ссылки, блоки кода,
картинки.

```md
У каждого P есть свой {{</* term "mcache" */>}}
Пер-P кэш мелких объектов. Аллокация из него идёт **без единого атомика** —
быстрый путь не трогает разделяемое состояние.
{{</* /term */>}}, и попадание в него — самый дешёвый путь аллокации.
```

| Параметр | Смысл |
|---|---|
| *позиционный 0* или `word=` | слово так, как оно читается в предложении (обязателен) |
| `title=` | шапка карточки, когда она отличается от слова; по умолчанию — само слово |
| `kind=` | свободный текст моно-ярлыка («структура рантайма», «железо»); без него ярлыка нет |
| `color=` | `accent` (по умолчанию — цвет бренда) \| `green` \| `copper` \| `blue` \| `gold` \| `red` |
| `href=` | адрес необязательной ссылки «читать подробнее»; внутренний путь разрешается в пермалинк |
| `more=` | текст этой ссылки (по умолчанию — i18n-ключ `term_more`) |

**Hugo не разрешает смешивать позиционные и именованные параметры.** Поэтому либо
короткая форма `{{</* term "mcache" */>}}`, либо полностью именованная — как
только понадобился `title=` или любой другой параметр, слово переезжает в
`word=`:

```md
данные лежат в {{</* term word="кэшах ядра" title="Кэш процессора"
                     kind="железо" color="green"
                     href="/inside/caches/" more="Разбор: иерархия памяти" */>}}
Три уровня — L1, L2, L3. Чем ближе к ядру, тем меньше и быстрее.
{{</* /term */>}}, а не в оперативке.
```

Две вещи, которые стоит знать:

- **Карточка рисуется не там, где ты её написал.** Абзац не может по правилам
  HTML содержать `<pre>` или `<div>` — парсер закроет его и разорвёт текст, — и
  поэтому шорткод оставляет в прозе инлайн-ссылку, а карточки собираются и
  выводятся после тела статьи. С выключенным JavaScript они там и видны, простым
  списком «Определения», и каждое слово ведёт к своему. Это и есть весь сценарий
  без JS — и то же самое уходит в печать.
- **Повторять термин ничего не стоит.** Идентификатор карточки — хеш от слова и
  определения, поэтому одно и то же слово с одним и тем же определением, взятое
  пять раз, делит одну карточку, и каждое упоминание её открывает. Изменишь
  определение — появится вторая карточка; если это не то, чего ты хотел, держи
  формулировку одинаковой.

### widget — `{{</* widget id="w-x" */>}}` {#widget}

Три формы (см. [Виджеты](#widgets)):

```md
{{</* widget id="w-escape" cap="Поиграй" note="— двигай слайдер, смотри на кучу" */>}}
```
- **самозакрытый, с `id`**: рендерит подпись и mount `.w-root`. Если у бандла
  есть ресурс страницы `widgets/w-escape/figure.html`, его содержимое заполняет
  mount как вид без JS; если такого ресурса нет, mount стартует пустым, и его
  заполнит `widgets/w-escape/widget.js`, как только сработает. `cap` перебивает
  подпись по умолчанию (i18n-ключ `widget_cap`), `note` — приглушённая ремарка
  после неё.
- **с `id` и телом внутри**: внутренний HTML — вид без JS вместо ресурса
  `figure.html`, для фолбэка, который короче держать рядом с шорткодом, чем в
  отдельном файле.
- **сырой** (без `id`, только тело): внутренний HTML кладётся прямо в фигуру.

### bigcard — `{{</* bigcard href="/playground/" k="Песочница →" t="…" s="…" */>}}` {#bigcard}

Ссылка-карточка: `k` — кикер, `t` — заголовок, `s` — подзаголовок. Внутренний
`href` разрешается в пермалинк (и потому переживает смену слага), внешний
проходит как есть.

### raw — `{{</* raw */>}}…{{</* /raw */>}}` {#raw}

Отдаёт своё содержимое HTML'ем как есть — для рукописных диаграмм ниже. В отличие
от голого HTML-блока в Markdown, шорткод захватывает тело целиком, поэтому
пустая строка внутри не обрывает блок.

### roadmap — `{{</* roadmap */>}}` {#roadmap}

Рисует блоки роадмапа — карточки «в работе», очередь, правила — из данных сайта:
либо плоский `data/roadmap.toml`, либо, на многоязычном сайте,
`data/roadmap/<lang>.toml` (так сделано в демо). Те же данные питают полосу «в
работе» на главной. Пример — `roadmap.md` в демо.

## Блоки кода {#code-blocks}

Обычные fenced-блоки. **Подсвечивается только Go** (Chroma на сборке,
перекрашенный в палитру темы); любой другой язык, ` ```text ` и голые заборы
рендерятся плоским `<code class="nohl">`. Два атрибута забора:

````md
```go {label="runtime/malloc.go (упрощено)" hl_lines=[2]}
func mallocgc(size uintptr) unsafe.Pointer { … }
```
````

- `{label="…"}` — подпись файла над блоком. Работает для любого языка.
- `{hl_lines=[2]}` — подсветка строк; диапазоны передаются строками:
  `hl_lines=[2,"5-7"]`. Только в Go-блоках (это опция Chroma).

## Ссылки {#links}

Внутренние ссылки пиши **контент-путями**, а не выходными URL: hook рендера
ссылок разрешает их через `site.GetPage` и переписывает в канонический пермалинк
— ссылка переживает смену слага, а на многоязычном сайте ещё и разрешается в
перевод на языке текущей страницы.

```md
[гайд про бандл](/inside/anatomy/guide-is-a-bundle)              ← контент-путь, разрешается
[его раздел](/inside/anatomy/guide-is-a-bundle#two-touches)      ← фрагмент сохраняется
```

Битая внутренняя ссылка **валит сборку** (`params.linkcheck = "error"`, по
умолчанию) или ругается в лог (`"warn"`).

Корневой путь с расширением — `/index.xml`, sitemap, статичная выгрузка — не
контент-страница, поэтому проверку по страницам он пропускает и уходит через
`relURL`. Именно это оставляет его живым на сайте, который лежит в подпути
(`https://example.github.io/taiga/`), где буквальный `/index.xml` указал бы в
корень домена и отдал 404.

## Диаграммы (вместо картинок) {#diagrams}

Картинок в гайдах у темы нет — диаграммы памяти собираются руками из HTML внутри
блока `raw`. Строительные детали (всё стилизует тема):

```md
{{</* raw */>}}
<div class="mem">
  <div class="mem-lab">заголовок слайса <span class="tot">→ 24 B</span></div>
  <div class="header">
    <div class="header-word ptr" data-tip="ptr · указатель"><span class="wk">ptr</span><span class="wv">→</span></div>
    <div class="header-word" data-tip="len · длина"><span class="wk">len</span><span class="wv">3</span></div>
  </div>
  <div class="mem-row"><div class="word live" data-tip="элемент">e0</div><div class="word spare" data-tip="запас">··</div></div>
</div>
{{</* /raw */>}}
```

- Ячейки `.mem`: `.word.live` / `.spare` / `.ro` / `.ptrcell` / `.dead`; ячейки
  заголовка — `.header-word.ptr` / `.tab`. Любой `data-tip` бесплатно получает
  тултип.
- Байтовая полоса: `.byte-strip > .byte-seg > .cells > .byte-box.{f0|f1|f2|pad}`
  плюс подпись `.seg-tag` (`.seg-tag.padtag` — для паддинга).
- Живой пример и того, и другого — страница `reference/kitchen-sink` в демо.

## Виджеты {#widgets}

Живая иллюстрация — это два касания, без правки шаблонов и `<head>`:

1. В тексте: шорткод `widget` (выше).
2. Рядом с `index.md`: папка `widgets/<id>/` — `widget.js` и, если готовых
   классов ниже не хватает, `widget.css`. Тема сама находит все
   `widgets/**.js` и `widgets/**.css` в бандле, склеивает их в порядке имён
   файлов, собирает и проставляет отпечаток и подключает результат **только
   на этой странице**.

Каждую фигуру регистрируй по id её mount'а, в `widgets/w-escape/widget.js`:

```js
Taiga.widget("w-escape", function (root) {
  // `root` — это mount .w-root; собирай интерактив здесь (ванильный JS).
});
```

`Taiga.widget` — крошечный рантайм внутри темы, и этот вызов — единственное,
что не поменяло форму: он всё так же ждёт DOM, находит mount по id и отдаёт
его твоему колбэку внутри `try/catch` — упавший виджет пишет в консоль и не
роняет соседей, а отсутствующий mount (шорткод убрали из текста) молча
пропускается. Id в шорткоде и в `Taiga.widget(…)` должны совпадать символ в
символ.

Бандл с более чем одним виджетом может завести папку `widgets/_shared/` для
кода или классов, нужных нескольким сразу, — `lib.js` (рантайм-guard плюс
общие хелперы) и `shared.css`. Сборка сортирует по имени файла, и `_shared`
всегда идёт раньше любой папки `w-*`, так что `_shared/lib.js` попадает в
склеенный скрипт первым, и его объявления уже видны каждому
`widgets/<id>/widget.js` после него — это фрагменты одной сборки, а не
отдельные файлы каждый со своим замыканием. Разобранный, с комментариями,
пример всей раскладки — статичный виджет, интерактивный и `_shared/` — лежит
на странице демо `reference/widget-anatomy`.

Пользуйся готовыми классами темы — тогда виджет выглядит родным без единой строки
своего CSS: `.w-row` (ряд контролов), `.w-btn` (`.primary`/`.ghost`), `.w-num`
(большое число; добавь `.tick` на его `<b>`, чтобы анимировать смену),
`.w-badge.ok`/`.alloc`, `.w-cap` (строка вывода), слайдеры — `.gc-slider-row` с
отсчётом `.nval`. Виджет должен работать офлайн, без зависимостей, и иметь
честное начальное состояние.

### Одна папка widgets/ на два языка {#widgets-i18n}

**Папка `widgets/` бандла общая для всех переводов.** У Hugo нет языковой
версии ресурса страницы: `index.md` и `index.ru.md` — две страницы *одного*
бандла, а папка `widgets/` у бандла ровно одна. Поэтому виджет с зашитыми
строками на половине двуязычного сайта покажет не тот язык — и ничто об этом
не предупредит.

В демо это решено ветвлением по языку документа, который тема ставит в
`<html lang>`:

```js
Taiga.widget("w-series-math", function (root) {
  var RU = document.documentElement.lang.indexOf("ru") === 0;
  var L = RU
    ? { onPart: "ты на части", cap: function (n, total) { … } }
    : { onPart: "you are on part", cap: function (n, total) { … } };

  // …собираем виджет, беря каждую видимую строку из L
});
```

Держи строки в одном объекте на язык в начале виджета и ниже не вставляй ни
одного литерала — вся дисциплина в этом. `indexOf("ru") === 0`, а не `=== "ru"`,
чтобы `languageCode` вида `ru-RU` тоже совпал. Полный образец —
`exampleSite/content/inside/anatomy/series-knows-itself/widgets/w-series-math/widget.js`.

То же верно для любого другого ресурса бандла: подменная `og.png`,
`figure.html` и всё, что лежит под `widgets/`, — по одной копии на оба языка.

## Обложки (OG-картинки) {#covers-og-images}

У каждого гайда есть Open Graph-обложка, которую тема рисует на сборке — подложка
плюс кикер серии, заголовок и минуты — **без единого действия автора** и на языке
страницы. Перебить для одного гайда: положить в его бандл `og.png` (или `og.jpg`)
либо задать `og_image:` во front matter. Сменить стиль для страницы — `og_style:`;
для всего сайта — `params.ogImages.style`
(см. [customizing.md](customizing.md#og-cover-styles)).

## Рубрики {#rubrics}

Рубрика — это секция с `_index.md`:

```yaml
title: "Приёмы"
params:
  label: "руководство"      # подпись кикера у гайдов этой рубрики и ссылка на карточке ленты
  slug_mono: "howto"        # моно-слаг в кикере самой страницы рубрики
  sub: "от установки до первого гайда."   # необязательный подзаголовок h1
  lead: "О чём эта рубрика."
  card_line: "Бери и поехали."            # необязательная строка на карточке рубрики на главной
  foot: "Заключительная заметка под рубрикой."
```

Перечисли секцию в `params.rubricSections` (конфиг сайта) — тогда её листовые
страницы получат полный макет гайда.

## Перевод гайда {#translating-a-guide}

Языковые версии связываются **суффиксом файла**, внутри одного бандла:

```
content/inside/anatomy/series-knows-itself/
  index.md        ← язык по умолчанию
  index.ru.md     ← перевод
  widgets/        ← общая для обоих (см. выше)
```

Индексные страницы рубрик и серий связываются так же: `_index.md` ↔ `_index.ru.md`.

Два поля обязаны быть **одинаковыми в обоих файлах**:

- **`slug`** — чтобы страница и её перевод делили путь URL
  (`/inside/anatomy/series-knows-itself/` и `/ru/inside/anatomy/series-knows-itself/`).
  Переключатель языка тогда прыгает между ними на месте, а не выбрасывает читателя
  на главную.
- **`tags`** — чтобы чипы, облако тегов и лента `/tags/#<tag>` сходились с обеих
  сторон. Переведёшь тег — получишь два полупустых облака.

Всё остальное — `title`, `description`, `lead`, `rail_title`, текст — переводится.
`weight`, `mins` и `version` структурные: копируй как есть.

Контент-пути во front matter (`related:`) и в ссылках разрешаются **по языку
страницы**, поэтому пишутся одинаково в обоих файлах: `related:
["inside/anatomy/palette-is-a-file"]` из `index.md` возьмёт английскую страницу, а из
`index.ru.md` — русскую.

Со стороны темы больше ничего не нужно: переключатель языка появляется сам,
альтернативы `hreflang` проставляются, а дата в ленте собирается из i18n-названий
месяцев — отдельный партиал на язык зеркалить не надо.

## Голос {#voice}

Эталон — сами гайды демо (`exampleSite/content/`). Коротко:

- Разговорно, точно, на «ты». Без корпоративного тона и рекламы.
- Дуга — **наивно → больно → починили → а вот как оно на самом деле**: сперва
  простая модель, потом место, где она ломается, потом настоящий механизм.
- Одна сквозная линия на гайд; возвращайся к ней.
- Курсивные чекпоинты после больших блоков: *«Теперь ты можешь …»*.
- Короткие бытовые метафоры (библиотечный формуляр, бирка на посылке).
- Виджеты вплетены конкретной задачей: «Поиграй — …».
- Заголовки `h2` — короткие и по делу, а не «Введение» и «Заключение».
- В конце — «Что дальше» и мост к следующей части.

При переносе текст копируется дословно — не правь молча; нашёл настоящую ошибку —
почини и скажи об этом.
