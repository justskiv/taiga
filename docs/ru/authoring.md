# Как писать гайд

Гайд — это **папка-бандл**: каталог с `index.md` и, если в тексте есть живые
иллюстрации, соседним `widgets.js`. Эта страница — контракт на то, как его
написать: front matter, шорткоды, код, ссылки, диаграммы, виджеты, обложки,
переводы и голос.

## Начни с архетипа {#archetype}

```sh
hugo new content howto/my-guide --kind guides
```

Получишь каркас бандла — `index.md` с заполненными основными полями и заготовку
`widgets.js`. **В пути нет `/index.md`**: передавай папку бандла, иначе Hugo
переключится в режим одиночного файла и молча не скопирует соседние файлы
архетипа. Папка без `widgets.js` — первый признак этой ошибки.

Первый сегмент пути должен быть секцией из `params.rubricSections` — именно это
делает страницу гайдом (две панели, кикер, чипы меты, оглавление, мост серии),
а не обычной колонкой.

## Front matter {#front-matter}

```yaml
title: "Значения и layout: как Go-значение лежит в памяти"
slug: memory-1-layout        # фиксирует URL (…/memory-1-layout/)
date: 2026-03-07             # лента, RSS, sitemap — в тексте статьи не появляется
description: "Одно предложение — для ленты, поиска и meta."
lead: "Лид-абзац (может быть длиннее description)."
series: ["memory"]           # членство в серии; у одиночного гайда — опустить
series_weight: 1             # порядок внутри серии
tags: [memory, layout, unsafe]
mins: 12                     # чип «~12 мин» — руками, а не .ReadingTime
version: "go1.26"            # свободный чип «проверено на»; без него — versionDefault
rail_title: "Layout"         # необязательно: 2–4 слова для левой панели и миникарты
related: []                  # для одиночных гайдов: 3–5 контент-путей для панели «ещё по теме»
```

| Поле | Обязательно | Что делает |
|---|---|---|
| `title` | да | Режется по первому `": "` на `<h1>` и `.sub`; переопределяется через `sub:`, а заголовок без `": "` остаётся в одну строку. |
| `slug` | да | Фиксирует красивый URL; после публикации не меняй — и держи **одинаковым во всех переводах** (см. [Перевод гайда](#translating-a-guide)). |
| `date` | да | Порядок ленты и RSS, печатается на карточке в ленте. **В тексте статьи не появляется никогда.** |
| `description` | да | Карточка ленты, результат поиска, `og:description`, `<description>` в RSS. |
| `lead` | желательно | Вводный абзац. Он же запасной вариант для `og:description`. |
| `series`, `series_weight` | для серий | См. [Серии и вес](#series-and-weight). |
| `tags` | желательно | Чипы, ведущие на страницу тегов, на якорь `#<tag>`. Держи **одинаковыми во всех переводах**. |
| `mins` | желательно | Чип времени чтения, сумма длительности на странице рубрики и шкала моста серии; без него — `.ReadingTime`. |
| `version` | опционально | Свободный чип «проверено на», печатается как есть (`go1.26`, `PostgreSQL 17`). Без него — `params.versionDefault`; нет ни того, ни другого ⇒ чипа нет. |
| `arch` | опционально | Дописывается к чипу версии через `·`, например `64-бит`. |
| `interactive` | опционально | Добавляет третий чип меты — перечень того, что на странице интерактивно. |
| `interactive_label` | опционально | Подпись этого чипа; по умолчанию — i18n-ключ `meta_interactive`. |
| `rail_title` | опционально | Короткое имя для левой панели, миникарты и моста серии. По умолчанию — заголовок до первого `:`. |
| `linkTitle` | опционально | Короткий заголовок для списка частей на странице рубрики. `<h1>` и `<title>` не трогает. |
| `related` | только одиночные | 3–5 **контент-путей** (`inside/palette-is-a-file`, а не URL) для панели «ещё по теме». У гайда в серии игнорируется — там панель показывает части. |
| `placeholder` | опционально | См. [Заглушки](#placeholders). |
| `og_image`, `og_style` | опционально | См. [Обложки (OG-картинки)](#covers-og-images). |
| `sub` | опционально | Переопределяет автоматическое деление `title`. |
| `intro` | опционально | Markdown между чипами меты и оглавлением. |
| `foot` | опционально | Markdown в самом низу, под мостом серии. |
| `toc_labels` | опционально | Карта «id заголовка → короткая подпись», чтобы укоротить длинный `h2` в оглавлении: `toc_labels: {two-fields: "Два поля"}`. |
| `draft` | опционально | Штатный Hugo. Архетип ставит `draft: true`. |

`kicker:` и `headline:` читаются только на **не-гайдовых** страницах (о сайте,
роадмап, `/tags/`); у гайда кикер собирается из рубрики и серии, и задавать их
бессмысленно.

### Серии и вес {#series-and-weight}

Членство — это две строки: `series: ["memory"]` и `series_weight: N`. Порядок,
кикер «часть N из M», левая панель и нижний мост (сегменты по `mins`, «осталось
~N мин») считаются из них на сборке. Добавить часть — положить файл с этими
двумя строками. Серия из одной части рендерится как одиночный гайд: механика
включается начиная с двух.

Метаданные серии живут на странице термина `content/series/<id>/_index.md`, и имя
папки `<id>` — тот самый якорь, на который ведут кикер и мост:

```yaml
title: "Память"             # имя серии, идёт в кикеры и в блок на странице рубрики
description: "Одна строка для блока серии на странице рубрики."
weight: 10                  # порядок серий на странице рубрики
params:
  label: "память"           # короткое имя в кикерах (иначе — title в нижнем регистре)
  tagline: "3 части, от байта до GC"   # необязательная строка рядом с «серия ·»
```

### Заглушки {#placeholders}

Ненаписанный гайд можно выпустить заглушкой: полный front matter (лента живёт с
него), тело — одна строка, и `placeholder: true`. Такой гайд считается в ленте,
в облаке тегов и в структуре серии («часть N из M»), но **не попадает ни в RSS,
ни в поисковый индекс**. Дописал текст — снял флаг.

## Шорткоды {#shortcodes}

Их пять, и это весь набор.

### callout — `{{</* callout type="trap" */>}}` {#callout}

Четыре типа: `key` (главная мысль), `trap` (ловушка), `note` (историческая
сноска), `internals` (под капотом). Подпись берётся из i18n (`callout_key`,
`callout_trap`, `callout_note`, `callout_internals`), переопределяется через
`label=`. Тело — Markdown. По умолчанию `type` — `note`.

```md
{{</* callout type="key" */>}}
Заголовок слайса — три слова: указатель, длина, ёмкость.
{{</* /callout */>}}
```

### widget — `{{</* widget id="w-x" */>}}` {#widget}

Три формы (см. [Виджеты](#widgets)):

```md
{{</* widget id="w-escape" cap="Поиграй" note="— двигай слайдер, смотри на кучу" */>}}
```
- **пустой mount** (самозакрывающийся): подпись плюс пустой `.w-root`, который
  заполнит твой `widgets.js`. `cap` перебивает значение по умолчанию (i18n-ключ
  `widget_cap`), `note` — приглушённая ремарка после подписи.
- **статичный фолбэк** (`id` + тело): HTML лежит внутри mount'а, пока его не
  заменит JS, — и остаётся, если JS выключен.
- **сырой** (без `id`, только тело): HTML кладётся прямо в фигуру.

### bigcard — `{{</* bigcard href="/playground/" k="Песочница →" t="…" s="…" */>}}` {#bigcard}

Ссылка-карточка: `k` — кикер, `t` — заголовок, `s` — подзаголовок. Внутренний
`href` разрешается в пермалинк (и потому переживает смену слага), внешний
проходит как есть.

### raw — `{{</* raw */>}}…{{</* /raw */>}}` {#raw}

Отдаёт своё содержимое HTML'ем как есть — для рукописных диаграмм ниже. В отличие
от голого HTML-блока в Markdown, шорткод захватывает тело целиком, поэтому
пустая строка внутри не обрывает блок.

### roadmap — `{{</* roadmap */>}}` {#roadmap}

Рисует блоки роадмапа — карточки «в работе», очередь, правила — из данных сайта:
либо плоский `data/roadmap.toml`, либо, на многоязычном сайте,
`data/roadmap/<lang>.toml` (так сделано в демо). Те же данные питают полосу «в
работе» на главной. Пример — `roadmap.md` в демо.

## Блоки кода {#code-blocks}

Обычные fenced-блоки. **Подсвечивается только Go** (Chroma на сборке,
перекрашенный в палитру темы); любой другой язык, ` ```text ` и голые заборы
рендерятся плоским `<code class="nohl">`. Два атрибута забора:

````md
```go {label="runtime/malloc.go (упрощено)" hl_lines=[2]}
func mallocgc(size uintptr) unsafe.Pointer { … }
```
````

- `{label="…"}` — подпись файла над блоком. Работает для любого языка.
- `{hl_lines=[2]}` — подсветка строк; диапазоны передаются строками:
  `hl_lines=[2,"5-7"]`. Только в Go-блоках (это опция Chroma).

## Ссылки {#links}

Внутренние ссылки пиши **контент-путями**, а не выходными URL: hook рендера
ссылок разрешает их через `site.GetPage` и переписывает в канонический пермалинк
— ссылка переживает смену слага, а на многоязычном сайте ещё и разрешается в
перевод на языке текущей страницы.

```md
[гайд про бандл](/inside/guide-is-a-bundle)              ← контент-путь, разрешается
[его раздел](/inside/guide-is-a-bundle#two-touches)      ← фрагмент сохраняется
```

Битая внутренняя ссылка **валит сборку** (`params.linkcheck = "error"`, по
умолчанию) или ругается в лог (`"warn"`).

Корневой путь с расширением — `/index.xml`, sitemap, статичная выгрузка — не
контент-страница, поэтому проверку по страницам он пропускает и уходит через
`relURL`. Именно это оставляет его живым на сайте, который лежит в подпути
(`https://example.github.io/taiga/`), где буквальный `/index.xml` указал бы в
корень домена и отдал 404.

## Диаграммы (вместо картинок) {#diagrams}

Картинок в гайдах у темы нет — диаграммы памяти собираются руками из HTML внутри
блока `raw`. Строительные детали (всё стилизует тема):

```md
{{</* raw */>}}
<div class="mem">
  <div class="mem-lab">заголовок слайса <span class="tot">→ 24 B</span></div>
  <div class="header">
    <div class="header-word ptr" data-tip="ptr · указатель"><span class="wk">ptr</span><span class="wv">→</span></div>
    <div class="header-word" data-tip="len · длина"><span class="wk">len</span><span class="wv">3</span></div>
  </div>
  <div class="mem-row"><div class="word live" data-tip="элемент">e0</div><div class="word spare" data-tip="запас">··</div></div>
</div>
{{</* /raw */>}}
```

- Ячейки `.mem`: `.word.live` / `.spare` / `.ro` / `.ptrcell` / `.dead`; ячейки
  заголовка — `.header-word.ptr` / `.tab`. Любой `data-tip` бесплатно получает
  тултип.
- Байтовая полоса: `.byte-strip > .byte-seg > .cells > .byte-box.{f0|f1|f2|pad}`
  плюс подпись `.seg-tag` (`.seg-tag.padtag` — для паддинга).
- Живой пример и того, и другого — страница `reference/kitchen-sink` в демо.

## Виджеты {#widgets}

Живая иллюстрация — это два касания, без правки шаблонов и `<head>`:

1. В тексте: шорткод `widget` (выше).
2. Рядом с `index.md`: файл `widgets.js` (и, при желании, `widgets.css`). Тема
   сама найдёт его, минифицирует, проставит отпечаток и подключит **только на
   этой странице**.

Каждую фигуру регистрируй по id её mount'а:

```js
Taiga.widget("w-escape", function (root) {
  // `root` — это mount .w-root; собирай интерактив здесь (ванильный JS).
});
```

`Taiga.widget` — крошечный рантайм внутри темы. Он ждёт DOM, находит mount по id
и отдаёт его твоему колбэку внутри `try/catch`: упавший виджет пишет в консоль и
не роняет соседей, а отсутствующий mount (шорткод убрали из текста) молча
пропускается. Id в шорткоде и в `Taiga.widget(…)` должны совпадать символ в
символ.

Пользуйся готовыми классами темы — тогда виджет выглядит родным без единой строки
своего CSS: `.w-row` (ряд контролов), `.w-btn` (`.primary`/`.ghost`), `.w-num`
(большое число; добавь `.tick` на его `<b>`, чтобы анимировать смену),
`.w-badge.ok`/`.alloc`, `.w-cap` (строка вывода), слайдеры — `.gc-slider-row` с
отсчётом `.nval`. Виджет должен работать офлайн, без зависимостей, и иметь
честное начальное состояние.

### Один widgets.js на два языка {#widgets-i18n}

**`widgets.js` бандла общий для всех переводов.** Никакого `widgets.ru.js` в Hugo
нет: `index.md` и `index.ru.md` — две страницы *одного* бандла, а ресурс
`widgets.js` у бандла ровно один. Поэтому виджет с зашитыми строками на половине
двуязычного сайта покажет не тот язык — и ничто об этом не предупредит.

В демо это решено ветвлением по языку документа, который тема ставит в
`<html lang>`:

```js
Taiga.widget("w-series-math", function (root) {
  var RU = document.documentElement.lang.indexOf("ru") === 0;
  var L = RU
    ? { onPart: "ты на части", cap: function (n, total) { … } }
    : { onPart: "you are on part", cap: function (n, total) { … } };

  // …собираем виджет, беря каждую видимую строку из L
});
```

Держи строки в одном объекте на язык в начале виджета и ниже не вставляй ни
одного литерала — вся дисциплина в этом. `indexOf("ru") === 0`, а не `=== "ru"`,
чтобы `languageCode` вида `ru-RU` тоже совпал. Полный образец —
`exampleSite/content/inside/series-knows-itself/widgets.js`.

То же верно для любого другого ресурса бандла: подменная `og.png` и `widgets.css`
— это тоже по одному файлу на оба языка.

## Обложки (OG-картинки) {#covers-og-images}

У каждого гайда есть Open Graph-обложка, которую тема рисует на сборке — подложка
плюс кикер серии, заголовок и минуты — **без единого действия автора** и на языке
страницы. Перебить для одного гайда: положить в его бандл `og.png` (или `og.jpg`)
либо задать `og_image:` во front matter. Сменить стиль для страницы — `og_style:`;
для всего сайта — `params.ogImages.style`
(см. [customizing.md](customizing.md#og-cover-styles)).

## Рубрики {#rubrics}

Рубрика — это секция с `_index.md`:

```yaml
title: "Приёмы"
params:
  label: "руководство"      # подпись кикера у гайдов этой рубрики и ссылка на карточке ленты
  slug_mono: "howto"        # моно-слаг в кикере самой страницы рубрики
  sub: "от установки до первого гайда."   # необязательный подзаголовок h1
  lead: "О чём эта рубрика."
  card_line: "Бери и поехали."            # необязательная строка на карточке рубрики на главной
  foot: "Заключительная заметка под рубрикой."
```

Перечисли секцию в `params.rubricSections` (конфиг сайта) — тогда её листовые
страницы получат полный макет гайда.

## Перевод гайда {#translating-a-guide}

Языковые версии связываются **суффиксом файла**, внутри одного бандла:

```
content/inside/series-knows-itself/
  index.md        ← язык по умолчанию
  index.ru.md     ← перевод
  widgets.js      ← общий для обоих (см. выше)
```

Индексные страницы рубрик и серий связываются так же: `_index.md` ↔ `_index.ru.md`.

Два поля обязаны быть **одинаковыми в обоих файлах**:

- **`slug`** — чтобы страница и её перевод делили путь URL
  (`/inside/series-knows-itself/` и `/ru/inside/series-knows-itself/`).
  Переключатель языка тогда прыгает между ними на месте, а не выбрасывает читателя
  на главную.
- **`tags`** — чтобы чипы, облако тегов и лента `/tags/#<tag>` сходились с обеих
  сторон. Переведёшь тег — получишь два полупустых облака.

Всё остальное — `title`, `description`, `lead`, `rail_title`, текст — переводится.
`series`, `series_weight`, `mins` и `version` структурные: копируй как есть.

Контент-пути во front matter (`related:`) и в ссылках разрешаются **по языку
страницы**, поэтому пишутся одинаково в обоих файлах: `related:
["inside/palette-is-a-file"]` из `index.md` возьмёт английскую страницу, а из
`index.ru.md` — русскую.

Со стороны темы больше ничего не нужно: переключатель языка появляется сам,
альтернативы `hreflang` проставляются, а дата в ленте собирается из i18n-названий
месяцев — отдельный партиал на язык зеркалить не надо.

## Голос {#voice}

Эталон — сами гайды демо (`exampleSite/content/`). Коротко:

- Разговорно, точно, на «ты». Без корпоративного тона и рекламы.
- Дуга — **наивно → больно → починили → а вот как оно на самом деле**: сперва
  простая модель, потом место, где она ломается, потом настоящий механизм.
- Одна сквозная линия на гайд; возвращайся к ней.
- Курсивные чекпоинты после больших блоков: *«Теперь ты можешь …»*.
- Короткие бытовые метафоры (библиотечный формуляр, бирка на посылке).
- Виджеты вплетены конкретной задачей: «Поиграй — …».
- Заголовки `h2` — короткие и по делу, а не «Введение» и «Заключение».
- В конце — «Что дальше» и мост к следующей части.

При переносе текст копируется дословно — не правь молча; нашёл настоящую ошибку —
почини и скажи об этом.
