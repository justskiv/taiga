# Adding a language

The theme is language-agnostic: adding a language is **content + config + an
i18n file**, with no template edits. The config already ships in multilingual
shape (`[languages.ru]`), so a second language is additive, not a migration.
English UI strings ship too (`i18n/en.toml`), so English needs no string work.

## Steps

### 1. Declare the language

Add a `[languages.<code>]` block to your site config. The default language has no
path prefix; others get `/<code>/…`.

```toml
defaultContentLanguage = "ru"

[languages.ru]
  languageCode = "ru"
  languageName = "Русский"
  weight = 1

[languages.en]
  languageCode = "en"
  languageName = "English"
  weight = 2
  [languages.en.params]
    heroLine = "guides on Go internals and practice"
    heroSat  = "· a new guide every Saturday"
  [[languages.en.menus.main]]
    name = "How-to"
    pageRef = "/howto"
    weight = 10
  # … the rest of the menu
```

Per-language `params` (hero, footer tag) and `menus` override the defaults for
that language.

### 2. Translate content by file suffix

Put the translation next to the original with a language suffix — Hugo links them
automatically (`.Translations`):

```
content/howto/get-started/index.md       # ru (default)
content/howto/get-started/index.en.md    # en
```

This applies to rubric `_index.md` and series-term `_index.md` too. (For many
translations, a per-language `contentDir` with parallel trees is the documented
alternative.)

### 3. Provide UI strings

If the language isn't Russian or English, add `i18n/<code>.toml` to your site
with the keys from the theme's `i18n/ru.toml` (67 of them). Hugo merges yours
over the theme's, so a partial file is fine — untranslated keys fall back.

### 4. Two theme-side touches

- **Feed dates.** `date-ru.html` writes Russian month abbreviations. For a
  non-Russian language, mirror `layouts/_partials/date-ru.html` in your site with
  the month names (or `time.Format`) for that language.
- **Language switcher.** The theme ships no switcher yet (there's one language).
  When you add a second, add a small partial that iterates `.Translations` and
  render it in a header/footer hook or an overridden `header.html`. It should
  only appear when `.IsTranslated`.

### 5. Nothing hardcoded

Templates already use i18n keys and `.Language` data only — `<html lang>` and
`og:locale` follow the page language, and `hreflang` alternates emit once there's
more than one language. You won't find a Russian string or a hardcoded `lang` to
chase.

## CJK note

Priority future languages are `en` and `ja` (`zh` likely). CJK works the same way
but needs three extra assets you provide:

- **Fonts.** Inter/JetBrains Mono carry no kana/kanji/hanzi — ship Noto (or
  similar) subsets for that language, both for body text and for OG covers
  (a separate TTF in your `assets/og/<style>/`).
- **Pagefind.** It builds a per-language index by page `lang` automatically; the
  CJK segmenter needs Pagefind's `extended` build (the standard build covers
  Russian and Latin stemming).
- **i18n.** An `i18n/<code>.toml` as in step 3.
