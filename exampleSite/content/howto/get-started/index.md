## Что это за тема {#what-it-is}

Тема для учебных гайдов, которые читают подряд, а не вразброс. Контент раскладывается в три уровня: рубрики → серии → гайды. Серия объявляется двумя полями front matter; всё остальное — «часть N из M» в кикере, панель-оглавление слева, мост с «осталось ~N мин» внизу — считается на сборке. Гайд — это папка: рядом с текстом лежат его интерактивные виджеты, стили и обложка, и переносится он целиком, одной папкой. Всё, что видно на этом демо, — лента, поиск, палитры, генерация обложек — штатные механизмы темы: демо написано ею о ней самой.

Из инструментов нужны два бинаря. Первый — обычный `hugo`, не extended: Sass в теме нет, а esbuild для JS встроен в стандартный бинарь. Второй — `pagefind`, полнотекстовый поиск по собранному сайту. Node, npm и `node_modules` не участвуют ни в сборке, ни в разработке. Минимальная версия Hugo объявлена в конфиге темы: слишком старый бинарь остановит сборку с внятной ошибкой, а не со странным поведением.

## Три способа установить {#three-ways}

Пути ниже — плейсхолдеры: подставь адрес репозитория темы.

**Hugo Module** — основной способ. В конфиг сайта:

```toml {label="hugo.toml — тема как модуль"}
[module]
  [[module.imports]]
    path = "github.com/user/theme"
```

Затем `hugo mod get -u` (если сайт ещё не модуль — один раз `hugo mod init github.com/user/site`). Версии резолвятся по semver-тегам темы; для локальной правки темы — `replace` в `go.mod` сайта.

**Git submodule** — если модули не хочется:

```bash {label="тема сабмодулем"}
git submodule add https://github.com/user/theme themes/theme
```

и `theme = "theme"` в конфиге. В CI не забудь checkout с `submodules: recursive`.

**Копия в `themes/`** — клон или распакованный архив в `themes/theme` плюс тот же `theme = "theme"`. Обновления руками; зато способ годится, чтобы просто посмотреть тему без обязательств.

{{< callout type="key" label="Главная мысль" >}}
Все три способа равноправны: тема держит ассеты в стандартных каталогах и не требует собирать саму себя. Разница только в том, как приезжают обновления.
{{< /callout >}}

## Минимальный конфиг {#minimal-config}

```toml {label="hugo.toml"}
baseURL = "https://example.org/"
title = "Мой учебник"
theme = "theme"                # не нужен при установке модулем
defaultContentLanguage = "ru"

[languages.ru]
  languageCode = "ru"
  languageName = "Русский"
  weight = 1

[taxonomies]
  tag = "tags"
  series = "series"

[[menus.main]]
  name = "Гайды"
  url = "/guides/"
  weight = 10
```

Блок `[languages.ru]` выглядит лишним, пока язык один, но он делает добавление второго языка аддитивным шагом, а не миграцией конфига. Остальное поведение настраивается через `[params]` — hero-строка главной, число видимых постов ленты, «гайд недели», палитра по умолчанию, — и у всего есть дефолты: сайт собирается и без единого параметра. Полный реестр — в `docs/params.md` темы.

## Структура контента {#content-layout}

```text {label="content/"}
content/
├── _index.md              # главная: hero-строка и данные
├── guides/
│   ├── _index.md          # рубрика: label, lead
│   └── my-first/
│       └── index.md       # гайд = папка (leaf bundle)
└── series/
    └── track/
        └── _index.md      # метаданные серии: title, description
```

Имена секций — свои: тема оперирует «секциями», «сериями» и «гайдами» и не хардкодит ни названий рубрик, ни предметных слов. Учебник про базы данных и учебник про акварель раскладываются одинаково.

## Собрать {#build}

```bash {label="разработка и прод"}
hugo server                        # разработка
hugo && pagefind --site public     # прод: сайт + поисковый индекс
```

Команд две, потому что pagefind строит индекс по уже собранному сайту. В dev без индекса поиск не ломается: модалка тихо подсказывает, какой командой он собирается.

*Теперь ты можешь развернуть сайт на теме с нуля и собрать его в один прогон — осталось его наполнить.*
