## Что это

taiga — тема для сайтов, которые **учат**: контент выстроен как рубрики → серии-курсы
→ гайды, а внутри гайда, посреди текста, могут жить живые JavaScript-виджеты.

Демо собрано на самой taiga, и каждый гайд на нём — про taiga: поэтому каждый
механизм работает прямо на той странице, которая его описывает. Страница сама себе
доказательство.

## Два бинарника — и всё

`hugo` и `pagefind`. Ни Node, ни npm, ни CDN, ни трекеров, ни сборочного конвейера,
за которым надо присматривать. Ванильный JS, собранный встроенным в Hugo esbuild,
обычный CSS, self-hosted woff2-шрифты, статический индекс поиска. Это ограничение —
не случайность, а суть дизайна: каждая фича ниже построена так, чтобы в него уместиться.

## Возможности

- **Интерактивные виджеты как контент.** Кладёте `widgets.js` рядом с `index.md`
  гайда — он находится, минифицируется и грузится только на этой странице. Один
  `{{< widget >}}` в тексте монтирует его, а рантайм `Taiga.widget` изолирует
  падения: сломавшийся виджет не утащит за собой соседей.
- **Битая внутренняя ссылка роняет сборку.** Render-хук проверяет каждую внутреннюю
  ссылку по реальным страницам (`linkcheck = "error" | "warn"`).
- **Обложки Open Graph без внешних сервисов.** `images.Text` рисует обложку каждому
  гайду на сборке — подложка, кикер серии, заголовок, минуты. Стили обложек — это
  *папки* (`assets/og/<style>/`); сайт добавляет или переопределяет свой без форка.
- **Полнотекстовый поиск без сервера.** Pagefind индексирует собранный сайт; ⌘K ищет
  слова *внутри* статей — со стеммингом и сниппетом, в собственной модалке темы.
- **Серия — полноценный механизм.** Серия — это папка внутри рубрики: одно это
  даёт лендинг серии, панель серии, кикеры «часть N из M» и нижний мост,
  отмасштабированный по времени чтения. Всё считается на сборке из одного
  источника правды.
- **Подсветка синтаксиса на сервере.** Go подсвечивается на сборке (Chroma,
  перекрашенная в палитру), с поддержкой `hl_lines`. Никакого клиентского
  подсветчика и никакой вспышки нестилизованного кода.
- **Палитры — это данные.** Каждая из семи палитр — один файл `data/themes/<id>.toml`;
  сайт добавляет свою одним файлом, и она появляется и в CSS, и в пикере тем.
  В комплекте тёмные, серая и светлая.
- **Двуязычность, и демо это доказывает.** Английский и русский идут в комплекте:
  строки интерфейса, даты, строки JS. Переключатель языков появляется сам, как только
  у сайта есть второй язык. Добавить третий — это контент, конфиг и один i18n-файл,
  без правки шаблонов.

## Требования

| | |
|---|---|
| **Hugo** | ≥ 0.146 (разрабатывается против v0.154.x). Сборка `extended` **не** нужна — тема на обычном CSS. |
| **[Pagefind](https://pagefind.app/)** | Для поиска. Второй шаг сборки — и по-прежнему без Node. |

## Установка

Выберите один способ.

**Модуль Hugo** (рекомендуется). Добавьте импорт в конфиг сайта — при импорте модуля
`theme` указывать **не** нужно:

```toml
[module]
  [[module.imports]]
    path = "github.com/justskiv/taiga"
```

Затем `hugo mod get -u`.

**Git-сабмодуль.** Добавьте в `themes/` и пропишите `theme = "taiga"`:

```sh
git submodule add https://github.com/justskiv/taiga.git themes/taiga
```

**Копией.** Склонируйте или скачайте в `themes/taiga` и пропишите `theme = "taiga"`.

## Быстрый старт

```sh
hugo new site mysite && cd mysite
# установите тему одним из способов выше, затем:
hugo new content howto/get-started --kind guides
hugo server
```

Запустить демо из клона этого репозитория:

```sh
hugo server -s exampleSite --themesDir ../..
```

Полная сборка, с поиском:

```sh
hugo -s exampleSite --themesDir ../.. --gc --minify
pagefind --site exampleSite/public
```
